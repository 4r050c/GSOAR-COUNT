# ChroniCount – SOC Shift Case Counter ✅

A lightweight JavaScript bookmarklet built for Security Operations Center (SOC) analysts to quickly calculate the number of incident cases created during each operational shift within Google Chronicle SOAR. ChroniCount automatically identifies and excludes automation-generated cases, providing a more accurate representation of analyst-created workload and shift activity.

Unlike manual counting or exporting data into spreadsheets, ChroniCount performs the calculation directly within your authenticated Chronicle SOAR session, allowing analysts to generate reliable shift statistics in seconds.

---

# Overview

SOC teams often need to report the number of incidents handled during every shift. While Chronicle SOAR records every case created, many of those cases are generated automatically by playbooks, integrations, or automated detection rules. Including these automated cases can significantly inflate productivity metrics and make shift reports inaccurate.

ChroniCount solves this problem by:

* Retrieving case information directly from Chronicle SOAR
* Categorizing cases into the appropriate SOC shift
* Detecting and excluding automation-generated cases
* Producing analyst-focused case counts
* Displaying results instantly within the browser

This allows analysts and team leads to generate consistent shift metrics without manual filtering or spreadsheet calculations.

# How It Works

1. Open Google Chronicle SOAR in your browser.
2. Navigate to the Cases page.
3. Click the **ChroniCount** bookmarklet.
4. Enter the requested parameters (such as date or shift range) if prompted.
5. The bookmarklet retrieves the relevant case data from the current page/session.
6. Each case is analyzed and categorized into the appropriate SOC shift.
7. Automation-generated cases are identified using predefined filtering logic.
8. Filtered cases are excluded from the final analyst count.
9. A summary report is displayed showing:

   * Total cases
   * Automation-generated cases
   * Analyst-created cases
   * Shift-wise breakdown
